Building a Development Environment

Introduction

In an effort to get better control over the test environments some clean up of existing environments is being done and new environments will be created that will be assigned to people as needed.

The new environments are set to use a view "NONE", so you will get an error the first time you try to use them. Just set the view in the setContext file and re-login.

You should assume that the environment is in an unknown state and rebuild it. The steps for building a development environment are detailed below.

Steps for Building a Development Environment

1. Clean database by using dropAllTables command.

2. Change directory to /vobs/dte/domain/sql

3. Update your environment by executing ". environ"

4. Recreate tables by doing "clearmake -C gnu DB_USERID=<id> DB_PASSWORD=<pw> all"

5. Import test data by doing "clearmake -C gnu DB_USERID=<id> DB_PASSWORD=<pw> imp_test_data"

6. Update the configuration groups by doing "clearmake -C gnu DB_USERID=<id> DB_PASSWORD=<pw> PREFIX=<prefix> update_config"

7. Change directory to /vobs/dte/infrastructure/sql

8. Login to the database by using the "loginDb" command (will use id of test environment).

9. Run the script to the IFR tables by typing "@createIFR".

10. Run the script to create the ID Service tables by typing "@idService".

11. Change directory back to the running directory by "cd $RUN_DIR"

12. Start the IFR with the "runifr.sh" command.

13. Load the IFR with the "loadIFR" command (this is now checked into the infrastructure scripts directory).

14. Run the command.

You should now be fairly well set. If event channels have changed or you are not sure:

15. Start the channel admin with the "startCA" command.

16. Remove all existing event channels for the environment (no command to do this). You can use "trader -query EventChannel | grep $SBT_PREFIX" to get a list of the channels and then use the removeEC command to remove each channel.

17. Create all event channels with the "createSbtEC" command.

18. Create the ID service event channels with the "createIdServiceEC" command.
