For those who were unaware, the SBT_5.00.09 checkpoint is now using makefiles to build all idl. If you are familiar with how makefiles work you can look at the list of targets in the Makefile for a few more undocumented options. Described here are the basics that will allow you to build the idl.

Building the client IDL (this will automatically build the server idl)

1) Log into dte_dev

2) set your view

3) goto the client idl directory. Run this command -

clearmake -C gnu all

4) when this command is complete you will find the client idl zip files in the directory

/vobs/dte/client/idl/java_output

5) when this command is complete you will find the server idl zip files in the directory

/vobs/dte/server/idl/trc_output

6) You can check-in the resulting zips (both client and server) using this command.

clearmake -C gnu ci_zips (this command will generate output that looks like an error but is expected: see ci_zips output below ) 

Building the server IDL (this will not build the client)

1) Log into dte_dev

2) set your view

3) goto the server idl directory. Run this command.

clearmake -C gnu all

4) when this command is complete you will find the server idl zip files in the directory

/vobs/dte/server/idl/trc_output

5) You can check-in the resulting zips using this command.

clearmake -C gnu ci_zips (this command will generate output that looks like an error but is expected: see ci_zips output below ) 

Adding client IDL files to the build (when you add it here, the server will automatically use it)

1) goto the client idl directory and check-out and open the file client_idl.mk

2) add your file to the list (it should be obvious) 

Adding server IDL files to the build

1) goto the server idl directory and check-out and open the file server_idl.mk

2) add your file to the list (it should be obvious)
